Design v2 — Phone Indicator + Digits-Only Validation

Country selection drives both address and phone prefix. Phone field: digits only.

How it works

Selecting a country populates two things simultaneously:

  1. The country name field in the address block (e.g. "Colombia")
  2. The phone indicator prefix shown next to the phone input (e.g. "+57")

Country name and dial code stored as separate columns. Phone digits stored separately (no indicator prefix in value).

New DB columns (V060 migration)

-- Rename old single-string address
ALTER TABLE tenants RENAME COLUMN contact_address TO contact_street;

-- New address sub-fields
ALTER TABLE tenants ADD COLUMN contact_city VARCHAR(100) NOT NULL DEFAULT '';
ALTER TABLE tenants ADD COLUMN contact_state VARCHAR(100) NOT NULL DEFAULT '';
ALTER TABLE tenants ADD COLUMN contact_country VARCHAR(100) NOT NULL DEFAULT '';

-- Phone indicator (dial code, e.g. "57", "1", "34")
ALTER TABLE tenants ADD COLUMN contact_phone_indicator VARCHAR(10) NOT NULL DEFAULT '';

Frontend phone validation

Rule: /^\d+$/ — reject anything not digits. Error: "Phone must contain digits only."
Phone field disabled until country is selected (forces user to pick country first, ensuring indicator always set).
