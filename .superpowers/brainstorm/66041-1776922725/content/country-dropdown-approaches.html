Country Dropdown: Which Approach?

The only real implementation decision left — pick the one that fits

A

Custom React dropdown (Recommended)

A CountrySelect component built with useState. Renders a text input for search and a filtered list below. Static list of ~195 countries baked into a TypeScript constant. Colombia / Spain / USA pinned at the top via a separate array.

Pros

  • Zero new dependencies
  • Full control over styling
  • Matches existing Tailwind patterns in the codebase
  • Accessible (keyboard nav easy to add)

Cons

  • ~100 lines of component code to write
B

react-select library

Drop in react-select with a options array. Handles search, keyboard, and accessibility out of the box. Country list passed as options with pinned group at top.

Pros

  • Robust keyboard and accessibility
  • Very little code

Cons

  • Adds a new npm dependency
  • Default styling clashes with Tailwind — needs theme override
  • Larger bundle footprint
C

Native <select> + <datalist> search

A native HTML <select> for the dropdown and a separate <input> + <datalist> for search. Pinned countries added as first <optgroup>.

Pros

  • No JS needed for dropdown behavior
  • Native browser accessibility

Cons

  • Datalist search UX is inconsistent across browsers
  • Can't style the option list with Tailwind
  • The pinned-countries-at-top + separator pattern is awkward with optgroups
