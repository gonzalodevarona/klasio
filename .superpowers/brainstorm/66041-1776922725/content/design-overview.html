Full Design — Tenant Creation Form Overhaul

Two layers of changes: backend schema + domain model, then frontend form. Confirm each section looks right.

Backend Changes

1. Flyway Migration (V060)

ALTER TABLE tenants ALTER COLUMN contact_street SET NOT NULL;
ALTER TABLE tenants ADD COLUMN contact_city VARCHAR(100) NOT NULL DEFAULT '';
ALTER TABLE tenants ADD COLUMN contact_state VARCHAR(100) NOT NULL DEFAULT '';
ALTER TABLE tenants ADD COLUMN contact_country VARCHAR(100) NOT NULL DEFAULT '';

2. ContactInfo domain record

contactStreet, contactCity, contactState, contactCountry (all required) · contactPhone stays but now required · logo now required
4–9. Cascading updates (mechanical, no design decisions)
  • TenantController — 4 new @RequestParam (required), logo required, phone required

  • TenantResponseDto (infra DTO) — replace contactAddress
  • All tests updated to use new fixture shape

Frontend Changes

New CountrySelect component
web/src/components/tenants/CountrySelect.tsx
Static list of 195 countries in web/src/lib/countries.ts. Pinned group: Colombia · Spain · United States (shown first, separated by a divider). Text input filters list on keypress. Clicking item sets value + collapses dropdown. Closes on outside click (useRef + useEffect).
TenantForm changes
  • Button text: "Create Tenant"
  • Phone field: placeholder → "300 123 4567", validation: required + non-empty
  • Logo: validation added — required (null → error "Logo is required.")
  • contactAddress → 4 states: contactStreet, contactCity, contactState, contactCountry
  • Address block: Street (full width) + City/State (side by side) + CountrySelect
  • All 4 address fields validated: required + non-empty
  • FormData sends all 4 separate fields to API
Other files
  • tenants/new/page.tsx — title + h1 → "Create New Tenant"
  • types/tenant.ts — replace contactAddress with 4 fields
  • LogoUpload.tsx — add * to label
  • TenantDetail.tsx — display new address fields (street, city, state, country)

What's NOT changing

  • No phone format/regex validation beyond "required + non-empty" — just free text
  • Country stored as plain string (e.g. "Colombia") — not ISO code
  • No API versioning — same endpoint, new required params
